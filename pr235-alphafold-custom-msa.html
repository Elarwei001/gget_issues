2背景 / 问题Background / problem

Issue #52 由 kchennen 提出,诉求具体:希望能提交一份 MSA(而非一条原始序列)作为 gget alphafold 的输入。用例是:自动同源搜索会带进"预测错误、异构体、片段"等问题序列;传人工审编的干净 MSA 能提升折叠质量(引 PMID 33172385)。Issue #52 was filed by kchennen, a concrete request: to be able to submit an MSA (rather than a raw sequence) as input to gget alphafold. The use case: automatic homology search brings in problematic sequences (prediction errors, isoforms, fragments); a manually curated, clean MSA improves the fold (cites PMID 33172385).

底层需求:不是"换个输入格式"那么表面,而是把 MSA 构建的控制权交给用户——让懂行的人用自己精挑的同源集来折叠,这在结构预测里是公认能显著改善结果的做法。附带的实际好处:跳过 jackhmmer 就不必下载庞大基因库。Underlying need: not merely "another input format" but handing MSA construction control to the user — letting experts fold from their own hand-picked homolog set, a recognized way to materially improve structure prediction. A practical bonus: skipping jackhmmer avoids the huge genetic-database download.

3上下文:gget alphafold 平时怎么跑Context: how gget alphafold normally works

默认流程:序列 → jackhmmer 搜索多个远端基因库(uniref90 / smallbfd / mgnify…)拼出 MSA → 折叠。这一步要选"最近的镜像源"、流式下载分块的库(可达 ~2GB 临时文件),是整个流程里最重、最依赖网络的一段。本 PR 加的 msa 参数把这一整段短路掉:一旦提供 MSA,就完全不搜索、不下载,直接把用户 MSA 喂进后续折叠。The default flow: sequence → jackhmmer searches several remote genetic databases (uniref90 / smallbfd / mgnify …) to build an MSA → fold. That step picks the "nearest mirror" and streams chunked databases (up to ~2GB of temp files); it's the heaviest, most network-dependent part of the pipeline. This PR's msa argument short-circuits all of it: once an MSA is provided, there is no search and no download — the user's MSA feeds straight into the fold.

4自定义 MSA 输入 & 解析详解Custom MSA input & parsing

这是本 PR 最该看懂的技术核心。输入是一个 a3m 或对齐 FASTA 文件,第一条必须是 query(去掉 gap 后要逐字等于输入 sequence)。This is the technical heart of the PR. The input is an a3m or aligned-FASTA file, whose first sequence must be the query (equal to the input sequence once gaps are removed).

a3m 格式的大小写约定The a3m upper/lower-case convention

a3m 用大小写编码对齐信息:大写字母/gap(-)= 与 query 对齐的列;小写字母 = 相对 query 的"插入"(query 在此处没有对应残基)。解析要把这两类拆开:a3m encodes alignment with case: uppercase letters / gaps (-) = columns aligned to the query; lowercase letters = "insertions" relative to the query (positions the query has no residue for). Parsing separates the two:

# fixture 里的一条 homolog（带插入 "ef"）:
>sp|homolog3_with_insertion
MKVefLAAGSTKDEFGHIKLMN
        │└┴─ 小写 "ef" = 两个插入字符（不占对齐列）
        └─── 大写/gap 才是对齐列

parse_custom_msa() 产出三样东西,正是 AlphaFold 需要的 MSA 组件:parse_custom_msa() yields the three MSA components AlphaFold needs:

产出Output含义Meaning
aligned_sequences删掉所有小写插入后的序列(每行长度 = query 长度)sequences with all lowercase insertions removed (each row = query length)
deletion_matrix每个对齐列前"插入了几个字符"的计数(如 homolog3 在第 4 列前是 2)per aligned column, how many characters were inserted before it (e.g. homolog3 is 2 before column 4)
descriptions各序列的 headereach sequence's header
正确性核实(核心):这套"小写计入 deletion matrix + 用 str.maketrans 去小写得到对齐行"的逻辑,与 AlphaFold 自己的 parsers.parse_a3m 算法逐字一致,最后还包成 AlphaFold 原生的 alphafold.data.parsers.Msa 对象直接喂进流程——所以它不是"另写一套解析",而是复用 AlphaFold 的规范。对齐 FASTA 因为没有小写插入,deletion matrix 自然全 0。(我通过比对 AlphaFold 公开的 a3m 解析算法核实;本机无 AlphaFold 运行环境,未端到端跑折叠。)Correctness check (core): this "count lowercase into the deletion matrix + strip lowercase via str.maketrans to get aligned rows" logic is byte-for-byte identical to AlphaFold's own parsers.parse_a3m, and the result is wrapped in AlphaFold's native alphafold.data.parsers.Msa object fed straight into the pipeline — so it is not a re-implementation but a reuse of AlphaFold's convention. Aligned FASTA has no lowercase insertions, so its deletion matrix is all zeros. (Verified by comparing against AlphaFold's public a3m-parsing algorithm; no AlphaFold runtime here, so no end-to-end fold was run.)

8设计决策与权衡Design decisions & trade-offs

① 复用 AlphaFold 自己的 a3m 解析,而非另造Reuse AlphaFold's a3m parsing, don't reinvent

解析逐字对齐 AlphaFold 的 parse_a3m、并返回原生 Msa 对象——保证插入/删除语义和后续 make_msa_features 完全一致,避免"自己解析和 AlphaFold 理解不一致"的隐性 bug。正确的工程选择。the parsing matches AlphaFold's parse_a3m byte-for-byte and returns the native Msa object — guaranteeing the insertion/deletion semantics are identical to the downstream make_msa_features, avoiding subtle "our parser disagrees with AlphaFold" bugs. The right engineering choice.

② 强制首序列 = queryRequire the first sequence to equal the query

去 gap 后逐字比对首序列与输入 sequence,不一致就报错并把两条都打出来。这防住了"MSA 和 query 对不上"这类会悄悄产出错误结构的严重错误——好的防御。compares the first sequence (gaps removed) to the input sequence, erroring with both printed on mismatch. This guards against a serious "MSA doesn't match the query" error that would silently produce a wrong structure — good defense.

③ 先限 monomer,清晰报错Restrict to monomer first, with a clear error

multimer 自定义 MSA 涉及链间配对,复杂度高;先做 monomer、对多序列/多 MSA 明确报错,是稳妥的分步。a multimer custom MSA involves inter-chain pairing and is much more complex; shipping monomer first and erroring clearly on multi-sequence/multi-MSA is a sound incremental step.
