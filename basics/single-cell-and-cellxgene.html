📖 一句话入门One-line primer
你身体里几乎每个细胞都带着同一套 DNA,但一个神经元和一个免疫细胞长得、干的活天差地别——差别不在"有哪些基因",而在每个基因开多大(基因表达)。单细胞测序能一次量出成千上万个细胞、每个细胞里每个基因的表达量,拼成一张"细胞 × 基因"的大表。CZ CELLxGENE Census 把全世界这类数据集汇总成一个可查询的"细胞图谱";gget cellxgene 就是去这个图谱里按组织/细胞类型/基因取数据。 Almost every cell in your body carries the same DNA, yet a neuron and an immune cell look and behave completely differently — the difference isn't which genes they have, but how much each gene is turned on (gene expression). Single-cell sequencing measures, for thousands of individual cells, the expression of every gene in each one, forming a big "cells × genes" table. CZ CELLxGENE Census aggregates the world's such datasets into one queryable "cell atlas"; gget cellxgene pulls data from it by tissue / cell type / gene.

TL;DR:
  • 细胞是生命的基本单位;同一个体的细胞共享同一套基因组,靠"开哪些基因、开多大"分化成不同细胞类型(神经元、T 细胞…)。Cells are the basic unit of life; cells of one individual share one genome and differentiate into cell types (neurons, T cells, …) by which genes they turn on and how much.
  • 单细胞 RNA 测序(scRNA-seq)逐个细胞量出所有基因的表达,得到一张 细胞(行)× 基因(列) 的稀疏矩阵。Single-cell RNA-seq (scRNA-seq) measures every gene's expression in each cell, giving a cells (rows) × genes (columns) sparse matrix.
  • 把很多这样的数据集标准化、汇总,就是 CZ CELLxGENE Census —— 一个跨物种、可编程查询的细胞图谱。Standardizing and pooling many such datasets is CZ CELLxGENE Census — a cross-species, programmatically queryable cell atlas.
  • 它返回的数据用 AnnData 容器装:X=表达矩阵、obs=每个细胞的元数据(组织/细胞类型/疾病…)、var=基因信息、layers=同一矩阵的其它版本。Its data comes in an AnnData container: X = the expression matrix, obs = per-cell metadata (tissue / cell type / disease …), var = gene info, layers = alternate versions of the matrix.
  • gget cellxgene 让你一行命令按条件从这个图谱取 AnnData 或元数据。gget cellxgene pulls an AnnData or metadata from this atlas by your criteria in one command.

2为什么要看"单个细胞"Why look at single cells

早年的测序是"bulk(混样)":把一块组织里成千上万个细胞打碎、一起测,得到的是这群细胞的平均值。问题是——一块"肺组织"里其实混着上皮细胞、免疫细胞、血管细胞…… 平均值会把它们抹平,看不出"是哪种细胞、在干什么"。 Classic sequencing was bulk: grind up a piece of tissue (thousands of cells) and sequence them together, giving the average over that cell population. The problem — a piece of "lung" actually mixes epithelial, immune, vascular… cells, and the average blends them so you can't tell which cell type is doing what.

Bulk RNA-seqSingle-cell RNA-seq
测量单位Unit一群细胞的平均average over a population每个细胞各自一份each cell separately
能回答Answers"这块组织整体表达了啥""what the tissue expresses overall""有哪些细胞类型、各自表达啥、比例如何""which cell types are present, what each expresses, in what proportions"
一个比喻:bulk 像"把一锅汤搅匀尝一口";单细胞像"把汤里每一粒食材分开,逐个看清"。Analogy: bulk is "stir the soup and taste one spoon"; single-cell is "separate every ingredient and inspect each one".

3scRNA-seq:细胞 × 基因矩阵scRNA-seq: the cells × genes matrix

基因 → RNA → 蛋白:一个基因"开工"时会先转录成 mRNA。一个细胞里某基因的 mRNA 越多,通常说明这个基因在该细胞里"开得越大"。单细胞 RNA 测序把组织打散成一个个细胞,给每个细胞的 RNA 贴上独有的"条形码",再一起测序,最后按条形码把读到的 RNA 归回各自的细胞。结果就是一张大表: Gene → RNA → protein: when a gene is active it is transcribed into mRNA. The more mRNA of a gene in a cell, the more that gene is "on" there. scRNA-seq dissociates a tissue into single cells, tags each cell's RNA with a unique "barcode", sequences them together, and uses the barcodes to assign reads back to their cell of origin. The result is a big table:

            GeneA  GeneB  GeneC  ...   (每列一个基因 / one gene per column)
Cell_1        0      5      0    ...
Cell_2        3      0     12    ...
Cell_3        0      0      0    ...   (大多是 0 → 稀疏矩阵 / mostly zeros → sparse)
 ...
(每行一个细胞 / one cell per row)

这张 细胞 × 基因 表就是单细胞数据的核心。它通常很大(几万~几百万个细胞 × 两三万个基因)且稀疏(绝大多数格子是 0,因为任一细胞只表达一部分基因)。 This cells × genes table is the heart of single-cell data. It is usually large (tens of thousands to millions of cells × ~20–30k genes) and sparse (most entries are 0, since any one cell only expresses a subset of genes).

4同一套 DNA,细胞为何各不相同Same DNA, different cells

这是最关键的生物学直觉:你身上几乎所有细胞的 DNA 都一样(同一套基因组)。神经元、肌肉细胞、免疫细胞的不同,不来自"基因不同",而来自 基因表达的不同——每种细胞选择性地开一批基因、关另一批。 This is the key intuition: nearly all your cells share the same DNA (one genome). A neuron, a muscle cell, and an immune cell differ not because their genes differ, but because their gene expression differs — each cell type selectively turns some genes on and others off.

  • 细胞类型(cell type):相对稳定的身份,如"T 细胞""肺上皮细胞"。由一套特征基因(marker)的表达定义。Cell type: a relatively stable identity (e.g. "T cell", "lung epithelial cell"), defined by a set of marker genes it expresses.
  • 细胞状态(cell state):同一类型的暂时状态,如"激活的""正在分裂的""受感染的"。Cell state: a transient condition of the same type — "activated", "dividing", "infected".
  • 怎么认出类型:把成千上万个细胞按表达谱的相似度聚类,每一簇往往对应一种细胞类型;再看每簇高表达的 marker 基因给它命名。How types are found: cells are clustered by similarity of their expression profiles; each cluster usually maps to a cell type, named by its high marker genes.
所以单细胞数据不仅是一张表——它让你能问:"肺里有几种细胞?每种占多少?某个基因(如 ACE2)主要在哪类细胞里表达?"这类问题 bulk 答不了。So single-cell data isn't just a table — it lets you ask: "how many cell types are in the lung? in what proportions? which cell type mainly expresses a given gene (e.g. ACE2)?" — questions bulk can't answer.

5细胞图谱与 CZ CELLxGENE CensusCell atlases & CZ CELLxGENE Census

全世界实验室做了海量单细胞实验(不同组织、物种、疾病)。细胞图谱(cell atlas)的目标是把它们汇总成"人体(或小鼠…)所有细胞类型的参考地图"。但各家数据格式、基因命名、细胞类型标注五花八门,难以直接合用。 Labs worldwide have produced enormous numbers of single-cell experiments (many tissues, species, diseases). A cell atlas aims to combine them into a reference "map of all cell types" of the human (or mouse…) body. But datasets differ wildly in format, gene naming, and cell-type labels, so they can't be used together as-is.

CZ CELLxGENE(Chan Zuckerberg Initiative 主导)做两件事:①一个门户,把大量单细胞数据集标准化(统一的元数据字段、基因 ID、细胞类型本体)后集中托管、可在线浏览;②Census —— 把这些标准化数据整合成一个可编程查询的大对象:你不用下载整个数据集,而是"按条件切一块"(如"人 + 肺 + 某几种细胞类型 + 某几个基因")直接拿回内存。 CZ CELLxGENE (led by the Chan Zuckerberg Initiative) does two things: (1) a portal that standardizes many single-cell datasets (uniform metadata fields, gene IDs, a cell-type ontology) and hosts them for browsing; (2) the Census — those standardized data integrated into one programmatically queryable object: instead of downloading whole datasets, you "slice by criteria" (e.g. "human + lung + a few cell types + a few genes") straight into memory.

版本很重要:Census 有带日期的版本号(如 2023-05-15)。固定版本 → 查询结果可复现。这也是 gget cellxgene 有 census_version 参数的原因。Versioning matters: the Census has dated versions (e.g. 2023-05-15). Pin a version → reproducible queries. That's why gget cellxgene has a census_version argument.

6数据长什么样:AnnDataWhat the data looks like: AnnData

单细胞领域用一个标准容器 AnnData(annotated data,"带注释的数据")来装那张细胞×基因表 + 所有旁注。核心结构: The field uses a standard container, AnnData (annotated data), to hold the cells × genes table plus all its annotations. Core structure:

AnnData是什么What it is
X表达矩阵本身:行=细胞、列=基因(n_obs × n_vars)the expression matrix: rows=cells, cols=genes (n_obs × n_vars)
obs每个细胞的元数据表(一行一个细胞):组织、细胞类型、疾病、性别、供体…per-cell metadata (one row per cell): tissue, cell type, disease, sex, donor…
var每个基因的信息表(一行一个基因):基因 ID、基因名、长度…per-gene info (one row per gene): gene ID, gene name, length…
layers同一 X 的其它版本(如原始计数 vs 归一化后),键=层名alternate versions of X (e.g. raw counts vs normalized), keyed by layer name
obsm / uns细胞的降维坐标(如 UMAP)/ 杂项非结构化注释cell embeddings (e.g. UMAP) / miscellaneous unstructured annotations
串回我们刚修的 CI bug(PR #266 / issue #265):gget 的测试用 repr_dict() 把一个 AnnData "拍平"成结构字典(记录 obs/var/layers 各有哪些键)来和期望值比对。cellxgene-census 某次更新后,返回的 AnnData 的 layers 里多出一个键为 None 的空层,于是结构字典里凭空多了 layers: [None]、测试就挂了。修法:让 repr_dict 忽略 None 键(那是噪声,不是有意义的层)。—— 现在你知道 layers 在生物学上是什么了,这个 bug 也就不神秘了。 Tying back to the CI bug we just fixed (PR #266 / issue #265): gget's test uses repr_dict() to flatten an AnnData into a structure dict (which keys each of obs/var/layers has) and compare it to an expected value. After a cellxgene-census update, the returned AnnData's layers gained a spurious None-keyed empty layer, so the structure dict grew a bogus layers: [None] and the test broke. The fix: make repr_dict ignore None keys (that's noise, not a meaningful layer). Now that you know what layers means biologically, the bug isn't mysterious.

7gget cellxgene 做什么What gget cellxgene does

gget cellxgene 是 CZ CELLxGENE Census 的一个便捷接口:你给出筛选条件(物种、组织、细胞类型、基因、Census 版本…),它就从图谱里"切"出你要的那块,返回一个 AnnData(表达矩阵 + 注释),或只返回元数据(不含表达矩阵、更轻)。 gget cellxgene is a convenience interface to the CZ CELLxGENE Census: you give filter criteria (species, tissue, cell type, gene, Census version…) and it "slices" that piece out of the atlas, returning an AnnData (matrix + annotations) or just the metadata (no matrix, lighter).

# Python — 取一小块 AnnData
import gget
adata = gget.cellxgene(
    tissue="lung",
    cell_type=["mucus secreting cell", "neuroendocrine cell"],
    gene=["ACE2", "ABCA1", "SLC5A1"],
    census_version="2023-05-15",
)
# adata.n_obs 个细胞 × adata.n_vars 个基因;adata.obs 里有 tissue/cell_type 等
单细胞数据可以非常大:不加限制地取一个组织的全部细胞,可能是几十万行 × 两万列。务必用 tissue/cell_type/gene 收窄,或先只取 meta_only 元数据看看规模。 Single-cell data can be very large: an unrestricted pull of a whole tissue can be hundreds of thousands of rows × ~20k columns. Narrow with tissue/cell_type/gene, or fetch meta_only metadata first to gauge the size.
