2MSA(多序列比对):折叠的真正输入MSA: the real input to folding

MSA = Multiple Sequence Alignment(多序列比对)——把目标蛋白(query)和许多同源序列(不同物种里"同一个"蛋白的版本)按列对齐排成一张表,每一列是"进化上对应的同一个位置"。MSA = Multiple Sequence Alignment — the target protein (query) and many homologous sequences (versions of "the same" protein in different species) aligned column by column into a table, where each column is "the same evolutionarily corresponding position".

为什么折叠需要它:共进化Why folding needs it: co-evolution

如果蛋白里两个位置在三维空间里挨得很近、相互接触,那么进化中一个位置变了,另一个往往跟着变来维持接触——这叫共进化(co-evolution)。在 MSA 里看很多物种,就能发现"哪些列总是一起变"→ 反推出"这两个位置在 3D 里靠得近"→ 拼出结构。所以 MSA 越好(同源序列多样且干净),折叠越准。If two positions in a protein are close and touch in 3D, then in evolution when one changes the other often changes with it to keep the contact — this is co-evolution. Looking across many species in the MSA reveals "which columns always change together" → infers "these two positions are close in 3D" → builds the structure. So the better the MSA (diverse and clean homologs), the more accurate the fold.

🔗 类比:两个人如果总是成对出现、一起换衣服的颜色,你会猜他们关系很近。共进化就是从"位置总是成对变化"猜"它们在结构里挨着"。🔗 Analogy: if two people always show up together and change outfit colors in sync, you'd guess they're closely linked. Co-evolution infers "these positions sit next to each other in the structure" from "they always change in pairs".

3不同序列如何"按列对齐"How different sequences get aligned column-by-column

不同物种的同源蛋白长度不一、有插入有缺失,直接并排对不齐。序列对齐(alignment)就是:在合适位置插入 gap(空位,写作 -),让"同源的同一位置"落到同一列。Homologous proteins across species have different lengths, with insertions and deletions, so they don't line up as-is. Sequence alignment inserts gaps (written -) at the right places so that "the same homologous position" lands in the same column.

query : M K V L A A G S T
homo1 : M K V L A A G S T
homo2 : M K V L A A - S T   ← 这里缺一个残基，补 gap 对齐
        └────── 每一列 = 同源的同一个位置 ──────┘

对齐算法(动态规划 / profile 方法)会找"让整体最相似、gap 尽量少"的排法。对齐好之后,同一列才可比,共进化信号才读得出来。Alignment algorithms (dynamic programming / profile methods) find the arrangement that "maximizes overall similarity with as few gaps as possible". Only once aligned are columns comparable and the co-evolution signal readable.

5MSA 怎么存:a3m 与对齐 FASTAStoring an MSA: a3m & aligned FASTA

FASTA(与对齐 FASTA)FASTA (and aligned FASTA)

FASTA 是最基础的序列文本格式:每条序列一个 > 开头的标题行 + 若干行序列字母。对齐 FASTA(aligned FASTA) = 里面的序列已经对齐过(等长、用 - 补 gap),就是一份用普通 FASTA 存的 MSA。FASTA is the most basic sequence text format: a > header line + sequence lines per entry. Aligned FASTA = the sequences are already aligned (equal length, gaps as -) — an MSA stored as plain FASTA.

>query

>homolog1
MKVLAAGSTKDEF-HIKLMN   ← 对齐 FASTA：用 - 补 gap

a3m

a3m 也是 MSA 格式(HHsuite/AlphaFold 常用),像 FASTA 但用大小写编码对齐信息:a3m is also an MSA format (used by HHsuite/AlphaFold), like FASTA but encoding alignment with case:

  • 大写字母 / - = 与 query 对齐的列;uppercase / - = columns aligned to the query;
  • 小写字母 = 相对 query 的插入(query 在此没有对应残基)。lowercase = an insertion relative to the query (a position the query has no residue for).

MKVefLAAGSTKDEFGHIKLMN

6为什么要"自定义 MSA"Why supply a custom MSA

自动搜索省事,但会混进垃圾:片段、异构体、预测错误的序列……这些会稀释甚至污染共进化信号。自定义 MSA = 你(或懂行的人)手工审编一份只含高质量、真正相关同源序列的 MSA,交给 AlphaFold,以求更准的折叠。附带好处:跳过 jackhmmer 就不用下那 ~2GB 库,也让输入确定、可复现(固定 MSA vs 每次搜可能漂移的远端库)。Automatic search is convenient but pulls in junk: fragments, isoforms, mis-predicted sequences… which dilute or even pollute the co-evolution signal. A custom MSA = you (or an expert) hand-curate an MSA of only high-quality, genuinely-related homologs and hand it to AlphaFold for a better fold. Bonuses: skipping jackhmmer means no ~2GB download, and the input becomes deterministic and reproducible (a fixed MSA vs searching remote DBs that may drift).

为什么 AlphaFold 不能在 CI 里跑Why AlphaFold can't run in CI

CI(如 GitHub Actions)的机器小、CPU-only、有时间/磁盘限制;而跑一次 AlphaFold 需要:重依赖(alphafold 包、JAX、openmm、HMMER/jackhmmer 二进制)+ GPU(CPU 上慢到不可行)+ GB 级模型参数 + ~2GB 以上的基因库。这些在 CI 里装不下、跑不动、超时——所以相关 PR 的测试只能覆盖"可测的那部分"(如 MSA 解析/校验),没法端到端折叠。CI machines (e.g. GitHub Actions) are small, CPU-only, time- and disk-limited; a single AlphaFold run needs heavy deps (the alphafold package, JAX, openmm, HMMER/jackhmmer binaries) + a GPU (unusably slow on CPU) + GB-scale model parameters + 2GB+ of genetic databases. These won't install, won't run, and time out in CI — so tests for such PRs can only cover the testable part (e.g. MSA parsing/validation), not an end-to-end fold.
