📖 一句话入门One-line primer
一个"基因"不是只对应一个产物:细胞可以把它的零件(外显子)按不同方式拼装,得到多个"转录本"。UCSC 基因组浏览器把一段染色体上的各种信息——基因模型、变异、表达、调控、跨物种保守性——各画成一条横向"轨道",叠在同一坐标轴上。看懂这两件事,就看懂了 gget ucsc 背后的数据。 A "gene" doesn't map to a single product: the cell can assemble its parts (exons) in different ways to make several "transcripts". The UCSC Genome Browser draws every kind of information over one chromosomal region — gene models, variants, expression, regulation, cross-species conservation — as a horizontal "track" stacked on a shared coordinate axis. Understand these two ideas and you understand the data behind gget ucsc.

TL;DR: 转录本 = 基因经"转录 + 剪接"后得到的一条成熟 mRNA。同一基因的外显子可按不同方式拼接(可变剪接),所以一个基因有多个转录本、可能编码略不同的蛋白。不同数据库各自给转录本编号(Ensembl ENST、RefSeq NM_),MANE 从中挑一条当"代表"。UCSC 基因组浏览器把同一段坐标上的多类信息各画成一条轨道(track):基因模型、变异、表达、调控元件、保守性……gget ucsc 做的,就是把这些轨道的"搜索命中"整理成一张表。 Transcript = one mature mRNA produced from a gene via transcription + splicing. A gene's exons can be joined in different ways (alternative splicing), so one gene has many transcripts encoding slightly different proteins. Different databases number transcripts differently (Ensembl ENST, RefSeq NM_); MANE picks one "representative". The UCSC Genome Browser draws each kind of information over a region as a track: gene models, variants, expression, regulatory elements, conservation… gget ucsc turns those tracks' search hits into a tidy table.

2基因 → 转录本 → 蛋白Gene → transcript → protein

中心法则的简版:The short version of the central dogma:

DNA(基因) --转录 transcription--> 前体 RNA --剪接 splicing--> 成熟 mRNA(= 转录本) --翻译 translation--> 蛋白
  • 外显子(exon):基因里最终保留、拼进成熟 mRNA 的片段。Exon: the parts kept and joined into the mature mRNA.
  • 内含子(intron):夹在外显子之间、剪接时被切掉的片段。Intron: the parts between exons, cut out during splicing.
  • 转录本(transcript):一条剪接完成的成熟 mRNA。它才是"基因的一个具体版本"。Transcript: one finished, spliced mature mRNA — a concrete "version" of the gene.
  • UTR / CDS:转录本两端不编码蛋白的区段叫 UTR,中间编码蛋白的叫 CDS(编码序列)。在浏览器里 CDS 画成高方块、UTR 画成矮方块。UTR / CDS: the non-coding ends are UTRs; the protein-coding middle is the CDS. In the browser, CDS is a tall box, UTR a short box.

3为什么一个基因有多个转录本Why one gene has many transcripts

因为拼装方式不唯一——主要来自可变剪接(alternative splicing),外加不同的起始/终止位点: Because assembly isn't unique — mainly from alternative splicing, plus different start/end sites:

机制Mechanism结果Result
跳过某个外显子 / 保留某个内含子Skip an exon / retain an intron转录本长度、编码的蛋白不同Different length & encoded protein
用不同的转录起点(启动子)Alternative start (promoter)5′ 端不同Different 5′ end
用不同的终止/加尾位点Alternative poly-A site3′ 端不同Different 3′ end
这就是为什么在 gget ucsc("BRCA2") 里,knownGene 轨道会返回 44 条 match——BRCA2 在 GENCODE 里被注释了 44 个转录本异构体(isoform)。它们共享大部分外显子,但拼法/两端各有差异。 This is why gget ucsc("BRCA2") returns 44 matches from the knownGene track — BRCA2 is annotated with 44 transcript isoforms in GENCODE. They share most exons but differ in splicing/ends.

4转录本的多套 ID(为什么"看起来无规律")Multiple ID systems for transcripts

同一个转录本,在不同数据库里有不同编号。gget ucsc 的 ucsc_id 只是把命中轨道的原生 ID 原样带出来: The same transcript has different IDs in different databases. The ucsc_id field just carries each track's native ID verbatim:

ID体系 / 出处System / source

MANE(Matched Annotation from NCBI and EMBL-EBI):NCBI 与 Ensembl 共同为每个基因钦定一条代表转录本,且让它的 RefSeq 与 Ensembl 编号精确对应同一条序列。所以浏览器里 MANE 轨道只有一条,标签常写成 BRCA2 (NM_000059.4 / ENST00000380152.8)——同一转录本的两套编号并列。 MANE (Matched Annotation from NCBI and EMBL-EBI): NCBI and Ensembl jointly pick one representative transcript per gene, with matching RefSeq and Ensembl IDs for the same sequence. So the MANE track has a single entry, often labelled BRCA2 (NM_000059.4 / ENST00000380152.8) — the two IDs for one transcript, side by side.

5基因组浏览器:轨道是什么The Genome Browser: what a track is

把染色体坐标当成一条横轴(图上方的尺子)。每一类信息各画成一条横向的"轨道(track)",上下叠放,共享这条横轴。你在某个位置往下看,就是"这个位置上,各类注释分别说了什么"。基因模型轨道里: Treat chromosomal coordinates as a horizontal axis (the ruler at top). Each kind of information is drawn as a horizontal "track", stacked vertically, sharing that axis. Reading down a position tells you "what every annotation says here." In a gene-model track:

  • 每条横线 = 一个转录本;方块 = 外显子(高=CDS,矮=UTR),细线 = 内含子,线上的小箭头 = 转录方向(链的正负)。 Each line = one transcript; boxes = exons (tall=CDS, short=UTR); thin line = introns; arrows = direction of transcription (strand).

7连回 gget ucscBack to gget ucsc

gget ucsc("BRCA2") 打的是 UCSC 的 /search API。它在多条轨道里搜这个词,返回时按轨道分组(positionMatches)。模块把每条命中展平成一行: gget ucsc("BRCA2") hits UCSC's /search API. It searches many tracks and returns matches grouped by track (positionMatches). The module flattens each hit to one row:

track      ucsc_id             chrom  start      end        name
knownGene  ENST00000380152.8   chr13  32315508   32400268   BRCA2 (ENST00000380152.8)
hgnc       HGNC:1101           chr13  32315086   32400268   BRCA2
mane       ENST00000380152.8   chr13  32315508   32400268   BRCA2 (NM_000059.4 / ENST00000380152.8)
...

所以浏览器里那一条条轨道,对应 API 里的 track group,再对应 DataFrame 里的 track 列。常见基因命中几十条轨道(BRCA2 约 46 组),所以不加过滤会返回很多行——用 track/limit 收窄。 So each browser track corresponds to an API track group, which becomes the track column in the DataFrame. Common genes hit dozens of tracks (~46 for BRCA2), so filter with track/limit.
