5如何使用 & 下载注意How to use & download caveat

# 列实验的文件(建议先过滤!)
gget encode ENCSR000AKS --assembly GRCh38 --file_format bam --output_type alignments

# 过滤后再下载
gget encode ENCSR000AKS --file_format bam --download --out_dir ./enc

# 自由词搜索
gget encode CTCF --type Experiment --limit 5
下载会拉取"过滤后剩下的全部文件"。一个实验默认有 82 个文件(探针实测 ENCSR000AKS),单个 bam 就 ~1GB——encode("ENCSR000AKS", download=True) 不加过滤会下几十 GB。文档应强提示:下载前务必用 assembly / file_format / output_type 收窄。 Download fetches every file left after filtering. One experiment has 82 files by default (probed ENCSR000AKS), a single bam is ~1 GB — so encode("ENCSR000AKS", download=True) unfiltered pulls tens of GB. The docs should strongly tell users to narrow with assembly / file_format / output_type before downloading.
