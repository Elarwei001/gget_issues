4响应结构 & 返回值形态Response structure & return shape

/search 返回一个 JSON,核心是 positionMatches:一个数组,每个元素是一个轨道分组(track group),组内有 matches 数组。模块把它展平成一行一个 match。下面是实测(BRCA2/hg38)结构: /search returns JSON whose core is positionMatches: an array where each element is a track group with a matches array. The module flattens it to one row per match. Probed structure (BRCA2/hg38):

positionMatches: [                     # 46 track groups for BRCA2/hg38
  { name, trackName:"knownGene", description, vis,
    matches: [
      { position:"chr13:32315508-32400268",
        hgFindMatches:"ENST00000380152.8",   # ← ucsc_id (may be URL-encoded, e.g. HGNC%3A1101)
        posName:"BRCA2 (NM_000059.4 / ENST00000380152.8)",  # ← HTML entities
        description:"...", canonical, highlight } ] } , ... ]

Column来源 → 含义Source → meaning
tracktrackName —— 匹配来自哪个轨道(knownGene / hgnc / mane …)trackName — which track the match came from
ucsc_idhgFindMatches,经 unquote 解 URL 编码 → 如 HGNC:1101、ENST00000380152.8hgFindMatches, URL-decoded via unquote → e.g. HGNC:1101
chrom / start / endposition 拆解:chr13:32315508-32400268 → chr13, 32315508, 32400268(去逗号、转 int)split from position (commas stripped, ints)

gget 化到位:异构嵌套 JSON → 扁平、带清晰列名的 DataFrame(或 json=True 时给 list[dict])。这正是 gget 的价值:把数据源"统一成可访问的接口"。Well gget-ified: heterogeneous nested JSON → a flat, clearly-named DataFrame (or list[dict] with json=True). Exactly gget's value: a uniform, accessible interface over the source.
