MSigDB (gsea-msigdb.org, Broad Institute) is the original curator of these gene sets; Enrichr (maayanlab.cloud, Ma'ayan Lab) re-hosts many MSigDB collections as its own libraries with a simple HTTP API. Why Enrichr rather than MSigDB directly: (1) Enrichr has a clean programmatic API, whereas MSigDB downloads require registration + license acceptance; (2) it reuses gget's existing enrichr module with zero new deps; (3) it also unlocks many non-MSigDB libraries (KEGG/GO/WikiPathways…).

8设计决策与权衡Design decisions & trade-offs

① 走 Enrichr 而非 MSigDB 官网① Via Enrichr, not the MSigDB site

选：复用现成 Enrichr 端点。得：零新依赖、统一文本格式、覆盖上百文库（不止 MSigDB）、直接嵌进 enrichr 模块。弃：MSigDB 官方的版本元数据、systematic name、集描述、官方 .gmt 直发。划算——issue 只要"取基因集"，而不是完整 MSigDB 元数据门户。 Chosen: reuse the existing Enrichr endpoint. Gains: zero new deps, one uniform text format, hundreds of libraries (not just MSigDB), fits inside the enrichr module. Given up: MSigDB's official version metadata, systematic names, set descriptions, official .gmt feed. Worth it — the issue asked for "getting gene sets", not a full MSigDB metadata portal.

② 长格式 DataFrame 作默认返回② Long-format DataFrame as default

选：两列长表 (gene_set, gene)；json=True 给 dict。得：tidy、易 groupby/merge、与 gget 其它模块一致。弃：更紧凑但难处理的"一行一集+列表"宽格式。合理。 Chosen: two-column long table (gene_set, gene); dict via json=True. Gains: tidy, easy groupby/merge, consistent with other gget modules. Given up: the more compact but harder-to-process "one row per set + list" wide format. Reasonable.

③ 向后兼容的 CLI 改造③ Backward-compatible CLI reshaping

把 genes 从 nargs="+" 放宽到 nargs="*"、--database 从 required=True 改为可选，再在运行时按 --get_library 是否存在决定"必须有 genes+database"还是"取文库"。既不破坏老用法，又让新旗标共存。细节到位。 Loosens genes from nargs="+" to nargs="*" and --database from required=True to optional, then decides at runtime — based on whether --get_library is present — whether "genes+database are required" or "fetch library". Preserves old usage while letting the new flag coexist. Well handled.
